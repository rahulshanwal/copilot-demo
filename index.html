The Mental Model

Copilot is your "Super-Smart Intern"

Imagine a brilliant intern who has read every documentation page on the internet.

  • They know syntax perfectly.
  • They know patterns (React, Ember, Redux).
  • BUT: They don't know *our* business logic or *our* coding style or *our* architecture... unless you tell them.
The Golden Rule:
If you give an intern vague instructions, you get bad code.
If you give Copilot vague prompts, you get hallucinations.

MODES
04

How to Interact (The Interface)

Copilot lives in three places in your editor:

  • 1. Ghost Text (Inline): The grey text you see while typing. Best for boilerplate.
  • 2. Copilot Chat (Sidebar): A conversation. Best for architectural planning.
  • 3. Inline Chat (Cmd+I): Highlight code and ask for changes directly.
ENGINE
05

Under the Hood: The Lifecycle of a Prompt

1. Input & Context Gathering Copilot grabs relevant snippets from open tabs, cursor position, and recent history.
3. Tokenization Text is converted into "Tokens" (number representations). The LLM predicts the next numbers.
2. Prompt Construction It wraps your input (i.e the "user prompt") in a hidden "System Prompt" to guide the model.
4. Prediction The model (GPT-4o/ Claude/ Gemini) predicts the most likely response tokens and sends back responses.

PROMPTS
07

Prompt Engineering: The 3 S's

Garbage In, Garbage Out. How to talk so the machine listens.

1. Simple

Break complex tasks down. Don't ask for a whole app. Ask for the layout, then the logic, then the styles.

2. Specific

Be explicit. "Create a login form using React Hook Form and Zod."

3. Sample

Provide examples. "Make it look like the `Button.tsx` component."

CONTEXT
09

Context Engineering: Strategies

The #1 reason for bad answers is missing context.

Strategies for Better Context

  • The "Open Tab" Strategy: Keep the 3-4 most relevant files open. Copilot prioritizes these above all else.
  • Naming Hygiene: Use descriptive variable names (`isUserLoggedIn` vs `flag`).
  • Precision Context:
    • #file: Explicitly reference a file.
    • #selection: Focus strictly on highlighted code.
RULES
10

Scaling Context: `.instructions.md`

Don't repeat yourself. Automate your team standards.

# Project Standards (.github/copilot-instructions.md)
- All components must be Functional React Components.
- Use `const` over `let`.
- When migrating from Ember, preserve `data-test-id`.

Copilot will now always read this file before answering.

META
11

Meta-Prompting: Use AI to Build AI

Don't write the prompt. Ask Copilot to write the prompt.

The Request

"I want to migrate a folder of Ember components to React. 
Write me a perfect prompt I can give to the @workspace agent 
to do this in bulk, preserving our CSS modules."

The Result

Copilot generates a highly specific, paragraph-long prompt. Copy that output, and feed it back into the agent.
